TTRPG Wiki to Website Workflow

So, I've been working on a D&D campaign (as one does), and for it, I've been using Obsidian to work on worldbuilding and have been slowly building my own wiki for the world for the players to use (as one does).
Now, my initial idea was for my players to download Obsidian (along with Google Drive or OneDrive for snyc). However, I have recently learned something that is much, much better.

I have discovered a way to easily take an Obsidian Vault and turn it into a website through GitHub Pages.

And if you're interested in doing the same, let me explain (in the thread below)

Step 1: Write the Wiki


This is the hardest and longest step. But it's also the most fun. Download Obsidian from the link above and try it out. It's a text editor that keeps all of your files locally as .md files, which is basically a .txt file with Markdown formatting (the same formatting that Discord uses for italics, bold, and headings). If you want a view of how Obsidian could be use for D&D, check out this video (especially 03:59-16:36 and 22:43-27:51).

Step 2: Duplicate Your Vault

Vault = Main Obsidian Folder With All Your Files.
You don't technically need to do this step, but since the plug-in you're about to install is not made by Obsidian themselves, I would recommend this so that if something does go wrong, you still have all your files somewhere else. (For the record, I've never had anything go wrong in all my testing.)
If you have files that you don't want your players to see (possibly because there are spoilers), leave these files out of the duplication.

Step 3: Install the Webpage HTML Export Plug-In to Your Duplicated Wiki

It looks like this:

This is the key to everything.

Step 4: Configure the Plug-In

In the plug-in's settings, I would personally recommend turning off "Graph View" and "Backlinks" in Page Features

This is just a preference thing about the end product looks, so feel free to experiment.
Also in the 'Export Settings' tab, I would recommend turning on 'Use Relative Header Links' and 'Slugify Paths'.

Step 5: Choose Your Homepage

For the HTML to work, you'll need one file to be your 'homepage.' Figure out which page that will be, and then Add a Property to the file.

Call the Property 'title' and then copy and paste the file's name into the Property

Then, rename the file to 'index'.

Step 6: Test Run

Click the 'Export to HTML' button on the sidebar.

Select all the files you want to include in the website, set the Export Mode to 'Local Website', make sure 'Open after export' is on, decide where on in your files the local website will be, and then press 'Export'.

The program should open up your File Explorer to where your local website is (it will be a .HTML file.) If you double click it, you'll get a preview of what your website will look like once it's finished. Feel free to mess with the plug-in's settings to make your website look more like you want it to.

Step 7: Export Your Obsidian (For Real This Time)

Go back into the "Export as HTML" menu in your Obsidian wiki. Make sure the 'Export Mode' is set to 'Online Website' (You will also want to send the website into it's own subfolder in your File Explorer). Check the Settings wheel at the bottom to make sure all the settings are how you want them to be.

Step 8: Set Up GitHub

This is where things get a little complicated, but bare with me.
Go to GitHub's website and sign up for a free account or log into your own account. Create a New Repository, and name it however you'd like (your eventual website will be [username].github.io/[repository-name]). The visibility will need to be 'Public' (which it is by default).

Step 9: Upload Your Website

Click 'uploading an existing file.'

Then, drag and drop the HTML files into GitHub. Once its done uploading, scroll down to the bottom of the page and click the green 'Commit changes'.

Step 10: Activate Your GitHub Page

Go to 'Settings' for your Repository. Click on 'Pages' in the sidebar, then change the 'Source' to 'GitHub Actions'.

Once the page reloads, click 'Static HTML' from the options.

When the page changes, click 'Commit changes...' twice.

Wait a couple minutes.
Head back to the 'Pages' tab of 'Settings'.
You should see a box that says 'Your site is live at 'https://[username].github.io/[repository-name]/'. If you don't, just give it a few more moments, and then refresh the page.

Click the 'Visit site' to check out your new website!
